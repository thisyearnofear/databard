# ZerveHack Pitch

## Project Summary
DataBard starts with a question, not a blank notebook. Ask “What is the biggest risk hiding in this warehouse?” or “Which protocol health issue should we fix first?”, connect a data source, and the app executes the analysis end to end: fetches metadata, evaluates schema health, surfaces lineage and quality risks, generates a two-host script, and synthesizes it into a podcast-style answer.

The product turns structured analysis into something people will actually consume. Instead of burying insights in dashboards or SQL, DataBard packages them into a shareable episode with drill-down context and optional on-chain minting for Initia users.

What makes it fit ZerveHack is the workflow: the user brings the question, the agent executes the analysis, the app iterates through schema context and failures, and the final output is both understandable and deployable. The same codebase stays eligible for OpenMetadata because the core analysis still centers on metadata, tests, lineage, owners, PII, and freshness, and it stays aligned with Initia / DoraHacks because protocol teams can mint health attestations on-chain.

## Demo Narrative
1. Start on the homepage and enter a question.
2. Connect OpenMetadata or a protocol source like The Graph.
3. Pick a schema and show the analysis pipeline running.
4. Play the episode, highlighting the research question and the critical findings.
5. Open a segment to show drill-down context: columns, tests, lineage, owners, and PII.
6. If on the web3 path, connect an Initia wallet and mint the episode metadata.
7. Finish by showing the share link and OG card so the output feels productized, not just analytical.

## Track Fit
- **ZerveHack**: question-first analysis, agentic execution, end-to-end workflow.
- **OpenMetadata**: deep catalog integration and metadata-driven insights.
- **INITIATE / DoraHacks**: wallet auth, on-chain provenance, and minting on Initia.
